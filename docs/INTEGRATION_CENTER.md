# EASP 集成中心改造方案

> 定位：EASP 不是单纯 SSO 系统，而是企业 AI 能力接入层。业务系统只需把当前用户身份交给 EASP，即可在现有系统中嵌入懂权限、能调工具、可审计、可记忆的 AI 助手。

## 1. 三类接入入口

| 入口 | 面向对象 | 解决问题 | 主路径 |
|---|---|---|---|
| 应用接入 | 业务系统开发者 | 把 EASP AI 助手嵌入 CRM/OA/ERP/H5/PC 页面 | 创建应用 → 后端换 Token → 前端 iframe/SDK 嵌入 → 联调检测 |
| 身份源配置 | 企业管理员 | 员工使用企业微信/飞书/钉钉/OIDC 登录 EASP 控制台 | 选择身份源模板 → 填最少字段 → 测试登录 → 启用 |
| 业务工具接入 | 技术实施/开发者 | 把业务系统 API 变成 AI 可调用工具 | Connector → OpenAPI/REST 导入 → MCP Tool → 权限 → 审计 |

## 2. 应用接入是主路径

业务系统不要把 `app_secret` 暴露到前端。推荐流程：

```text
业务系统用户登录
  ↓
业务系统前端请求自己的后端 /api/easp/embed-token
  ↓
业务系统后端使用 app_id/app_secret + 当前 external_user_id 签名
  ↓
POST /api/v1/embed/token/exchange
  ↓
EASP 返回短期 easp-api-token
  ↓
业务系统前端用 iframe/SDK 加载助手
```

EASP 负责：

- 外部用户映射到 EASP 内部用户；
- 复用 EASP 角色、工具、Skill、MCP 权限；
- 调用业务工具前做权限判断；
- 审计固化 `source_app_id/external_system/external_user_id/user_uid`；
- 支持 Connector `user_token` 模式透传业务用户 Token。

## 3. 页面改造要求

### 应用接入页

应提供：

1. 接入应用列表；
2. App ID / App Secret 一次性展示与重置入口；
3. 允许域名、Token TTL、默认范围；
4. 「接入指南」Tab：自动生成后端换 Token、iframe、SDK 示例；
5. 「联调检测」Tab：检查应用状态、来源白名单、外部用户导入、默认角色、Token 换取前置条件；
6. 外部用户和审计日志入口。

### 身份源配置页

SSO 降级为“员工登录 EASP 控制台”的辅助能力。默认使用模板向导：企业微信、飞书、钉钉、自定义 OIDC/OAuth2。高级 OAuth 字段默认折叠。

当前实现：

- 后端提供 `/api/v1/tenants/:tenantId/sso/templates` 返回身份源模板、默认值、字段定义和高级字段标记；前端不硬编码模板业务字段。
- 前端页面名称为「身份源配置」，顶部明确说明该能力只用于员工登录 EASP 控制台，业务系统嵌入助手走「应用接入」。
- 模板入口包含企业微信、飞书、钉钉、自定义 OIDC/OAuth2；基础字段优先展示，高级 OAuth/接口字段默认折叠。
- 登录链接 `/sso/:tenantId` 独立展示，便于管理员复制给员工测试。

### 业务工具接入页

业务工具接入是增强能力：把内部 REST/OpenAPI/MCP 能力变成可治理工具。Connector 需要明确展示凭据模式：

- `static`：固定凭据；
- `user_token`：透传当前业务用户 Token；
- `none`：无认证。

若工具要求 `user_token` 但当前上下文无业务 Token，必须明确失败，禁止静默 fallback。

当前实现：

- 前端连接器页已升级为「业务工具接入」工作台，顶部展示四步流程：选择接入方式 → 配置凭据模式 → 导入工具 → 授权与审计。
- 支持三种接入入口卡片：MCP Server 接入、OpenAPI 导入、业务用户 Token 透传。
- 增加统计卡片：接入源、MCP Server、REST/OpenAPI、已导入工具。
- 接入源以卡片视图展示类型、传输、凭据模式、工具数量，并提供配置、工具管理、删除入口。
- MCP 工具页已升级为「工具导入与治理」工作台：选择导入方式 → 生成工具草稿 → 测试与发布 → 角色授权。
- MCP 工具页支持 OpenAPI 批量导入、REST 单接口导入、MCP Server 发现三种入口；顶部提供工具总数、生产可用、草稿/未启用、高风险统计。
- 角色管理页已升级为「授权工作台」：选择角色 → 菜单入口 → 工具与 Skill → 数据与限流。
- 授权工作台提供租户角色、系统角色、全功能角色、MCP授权项、Skill授权项统计；租户角色以卡片展示菜单/MCP/Skill授权数量，并保留角色明细表格。
- 角色编辑弹窗按基础信息、后台菜单入口、助手可调用能力、数据边界与调用频率分区，明确菜单权限与 MCP/Skill 后端执行授权的区别。
- 审计日志页已升级为「审计可观测」工作台：日志总数、嵌入来源、有外部身份、失败/拒绝、平均耗时统计。
- 审计页突出三段链路：外部来源/应用/用户 → EASP内部用户 → 工具或 Skill 的授权决策与执行结果。
- 审计详情抽屉按「身份链路」「工具 / Skill 执行」「请求上下文」分区展示，包含 App ID、外部系统、外部用户、User UID/User ID、结果/决策、耗时、Agent/Request ID、IP、User-Agent 和可复制 detail。
- 用量分析页已升级为「成本与调用可观测」工作台：统一查看模型 Token、缓存命中、工具/Skill 调用、来源归因、耗时和失败情况。
- 用量分析页新增总调用、缓存命中率、工具成功率、主来源、Top 来源、Top 模型、平均耗时、失败工具调用摘要；明细记录增加详情抽屉，可与审计日志通过来源、用户、Request ID、资源类型/资源ID交叉定位。
- 模型配置页已升级为「模型路由与健康」工作台：展示默认模型、启用供应商、启用模型、备用模型、模型路由摘要和配置流程。
- 模型配置页强调先测试连接再设为默认；默认模型异常时应明确报错或走后端受控 fallback，禁止静默 fallback 到错误模型。
- 记忆管理页已升级为「记忆治理」工作台：统一展示记忆池、长期记忆、来源状态、向量索引、召回解释和治理审计。
- 记忆治理页新增用户记忆、活跃记忆、向量索引率、记忆池统计；来源与状态卡片显示 Top 来源、active/archived/vector；治理链路卡片展示自动提取、敏感过滤、召回注入、混合检索开关。
- 用户记忆和治理审计支持详情抽屉，便于查看内容归因、content_hash、访问/出现/向量状态、脱敏前后预览和治理原因。
- API Key 管理页已升级为「凭证安全工作台」：展示 API Key 总数、可用凭证、高风险、从未使用、权限 scopes 覆盖率和轮换建议。
- API Key 页明确系统级凭证应放在业务系统后端或可信服务端，不暴露在 H5/小程序前端；完整密钥只创建时展示一次。
- API Key 安全详情抽屉展示凭证归属、权限边界、生命周期、最后使用、调用次数和轮换建议。
- 用户管理页已升级为「租户用户运营工作台」：展示租户用户、活跃用户、已注销、需关注账号、权限与角色、身份映射和账号生命周期。
- 用户列表新增状态/风险标签与最近登录信息；详情弹窗展示账号主体、登录情况、创建/删除时间、扩展属性和第三方身份关联。
- 用户页明确删除为软删除可恢复，角色分配只使用租户角色；外部用户/第三方身份用于映射与审计，权限主体仍是 EASP 内部用户。
- 租户管理页已升级为「租户运营与容量工作台」：展示租户总数、正常租户、到期风险、容量风险、用户容量、生命周期和配额治理。
- 租户列表新增运营风险标签和用户容量进度；详情抽屉展示租户主体、到期时间、当前用户数/用户上限、限流配额和运营建议。
- 新建/编辑租户弹窗明确提示创建租户会初始化管理员账号；状态、到期时间、用户上限和配额会影响登录、注册、创建用户和助手调用，后端最终校验。
- Skill 管理页已升级为「Skill 治理工作台」：展示 Skill 总数、生产可用、内置锁定、测试/草稿、生命周期、Schema/触发和执行编排。
- Skill 列表新增来源/内置锁定标签、生产就绪风险、步骤数、MCP 步骤数、输入字段数、使用次数和最近使用；详情弹窗展示 Skill 主体、来源、生命周期、风险、标签、编排 JSON 和输入 Schema。
- MCP 工具页新增工具治理详情抽屉，展示工具来源、连接器、生命周期、启用状态、锁定状态、风险等级、方法/路径、输入 Schema 和生产可用/高风险/未发布提示。
- MCP 工具详情抽屉新增「授权与可执行状态」分区：通过后端 `governance-status` 接口实时展示授权状态、已授权角色数、角色列表、当前用户是否可执行、命中角色和不可执行原因。
- 业务工具接入页新增接入源治理详情抽屉，展示接入源来源、内置锁定、类型、传输方式、凭据模式、认证类型、工具数、基础 URL、MCP Server、用户 Token 透传和治理建议。
- 内置/锁定资源保护已固化：Skill(created_by=system)、MCP Tool(is_builtin/locked)、Connector(is_builtin/locked/type=builtin) 不可删除、停用或普通编辑；前端禁用危险入口，后端 API 与助手治理工具均强制拦截。
- 导入/新建弹窗明确提示：导入只生成 MCP Tool，是否可被助手调用由生命周期、启用状态和角色授权共同决定。
- 保留接入源明细表格，便于运营查看完整字段。
- 新建/编辑弹窗明确说明：接入源只负责连接业务系统，真正暴露给助手的是 MCP 工具，需在工具管理中导入并授权。

## 4. 当前 UX 实现

- 应用接入页已升级为「接入工作台」：顶部展示四步接入流程（创建应用 → 后端换 Token → 前端嵌入 → 联调检测）。
- 新增应用统计卡片：接入应用数、启用中应用数、外部系统数。
- 应用列表增加卡片视图，主操作前置：接入指南、联调检测、外部用户、审计。
- 接入指南弹窗改为步骤化说明，并提供可复制的签名 Payload、iframe、JS SDK 示例。
- 联调检测弹窗增加检查项/已通过/待修复统计，失败项展示明确修复建议。
