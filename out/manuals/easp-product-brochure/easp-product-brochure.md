# EASP 产品推广手册

> Enterprise AI Service Platform｜企业级 API-to-MCP 智能服务平台  
> 版本：V1.0｜适用对象：企业管理者、IT 架构团队、AI 应用团队、渠道/销售团队

## 1. 一句话定位

EASP 是面向 B2B SaaS 场景的企业级智能服务平台，将企业已有 API 快速转换为可被 AI Agent 调用的 MCP 工具，并通过多租户、SSO、RBAC/ABAC、审计、记忆与 Skill 引擎，为企业提供安全、可治理、可运营的 AI 能力接入层。

## 2. 市场痛点

企业在落地 AI Agent 时，通常遇到四类阻碍：

- **系统多、接口散**：内部 API 分布在 CRM、ERP、OA、客服、数据平台等多个系统中，Agent 难以统一调用。
- **权限不可控**：模型能调用什么、谁能用什么工具、是否越权、是否跨租户，需要企业级治理能力。
- **集成成本高**：每接一个系统都要重复开发工具定义、鉴权、参数映射、错误处理与监控。
- **运营不可见**：模型调用了哪些工具、消耗了多少 Token、是否失败、谁发起的操作，缺少审计与分析。

## 3. EASP 的核心价值

EASP 用“API-to-MCP 网关 + 企业治理 + Skill 工作流 + 记忆系统”的组合，帮助企业把已有数字化资产变成 AI 可调用能力。

| 价值维度 | 说明 |
|---|---|
| 快速集成 | 导入 OpenAPI 后自动生成 MCP 工具，减少重复开发。 |
| 安全治理 | 多租户隔离、SSO、RBAC/ABAC 权限、API Key、审计日志形成闭环。 |
| 可运营 | 模型配置、用量分析、工具调用记录、Skill 执行历史可追踪。 |
| 可扩展 | 支持连接器、MCP 工具、Skill 引擎、记忆池、模型供应商扩展。 |

## 4. 产品架构概览

EASP 位于企业 API、AI 模型与业务用户之间，承担统一接入、权限控制、工具编排和审计治理职责。

```text
业务用户 / AI 助手 / 外部 Agent
          │
          ▼
EASP 企业级智能服务平台
  ├─ 认证与租户：SSO、JWT、多租户隔离
  ├─ 权限治理：RBAC/ABAC、角色功能权限、工具权限
  ├─ API-to-MCP：连接器、OpenAPI 导入、MCP 工具生成
  ├─ Skill 引擎：步骤编排、执行记录、Schema 驱动表单
  ├─ 记忆系统：记忆池、长期记忆、混合检索、治理审计
  ├─ 模型服务：多供应商、多模型、默认模型、流式对话
  └─ 运营审计：审计日志、用量统计、调用追踪
          │
          ▼
企业现有系统：CRM / ERP / OA / 数据库 / 内部服务 / 第三方 API
```

## 5. 核心能力

### 5.1 多租户与企业认证

- 租户隔离与租户生命周期管理。
- 租户到期、用户上限、状态校验。
- 标准登录与 SSO 登录自动回退。
- 支持后续对接 OAuth2、SAML、企业微信、钉钉、飞书等企业身份体系。

### 5.2 角色权限与功能菜单治理

- 系统级角色与租户级角色两层体系。
- 功能权限控制前端菜单与操作入口。
- 管理员可配置用户、角色、连接器、MCP 工具、Skill、记忆、模型配置、审计、API Key 等模块权限。
- 权限策略由后端统一判定，前端负责展示与交互。

### 5.3 API-to-MCP 企业网关

- 通过连接器管理企业 API 接入。
- 支持导入 OpenAPI v2/v3 规范并生成 MCP 工具。
- MCP SSE 连接、消息处理、工具调用、参数映射与响应转换。
- 让 AI Agent 以标准 MCP 协议调用企业能力。

### 5.4 Skill 引擎

- 支持创建、管理和执行 Skill。
- 以 Schema 驱动执行表单，降低业务人员使用门槛。
- 记录每次 Skill 执行详情、步骤耗时与结果。
- 可作为标准化业务流程、自动化运维、客服处理、数据查询任务的编排层。

### 5.5 AI 助手与上下文协同

- 支持 SSE 流式对话、Tool Calling、模型信息展示。
- 内置用户、角色、连接器、MCP 工具、租户等管理工具。
- 支持悬浮助手与独立助手页面，共用统一请求协议。
- 执行过程可追踪，默认简洁展示，展开查看完整轨迹。

### 5.6 记忆系统与治理

- 支持记忆池、记忆条目、用户长期记忆。
- 写入去重、敏感过滤、审计日志、自动提取/召回开关。
- 支持衰减评分、合并策略、混合检索融合与召回解释。
- 可用于个性化服务、长期上下文、企业知识沉淀与 Agent 经验复用。

### 5.7 模型配置与用量运营

- 支持多模型供应商、多模型配置、默认模型切换。
- 禁止静默 fallback 到错误模型配置，问题显性化。
- 支持模型调用、Token 消耗、API 用量基础统计。
- 为成本治理、性能优化、租户计费和运营分析提供数据基础。

### 5.8 审计、安全与可追踪

- 操作审计、工具调用审计、Skill 执行记录。
- 记录发起人、租户、Agent ID、耗时、IP、User-Agent 等上下文。
- 支持企业对 AI 调用链路做安全追踪与责任定位。

## 6. 典型应用场景

| 场景 | EASP 如何提供价值 |
|---|---|
| 企业内部 AI 助手 | 统一接入内部 API，按用户角色开放可调用工具。 |
| 客服/售后 Agent | 调用订单、工单、知识库、客户信息接口，实现自动处理。 |
| 销售运营助手 | 查询客户、线索、合同、回款等系统数据，辅助销售决策。 |
| IT 运维自动化 | 将监控、发布、工单、资产接口封装为受控 MCP 工具。 |
| 企业知识与记忆 | 将偏好、历史处理经验、知识片段沉淀为可治理记忆。 |
| SaaS 平台 AI 化 | 为 B2B SaaS 提供租户级 AI 能力开放、权限与用量运营。 |

## 7. 差异化优势

1. **不是单一聊天机器人**：EASP 关注企业 API 能力治理，让 AI 真正接入业务系统。
2. **不是简单 API 网关**：EASP 面向 AI Agent 调用场景，内置 MCP、Skill、记忆与模型路由能力。
3. **后端统一决策**：登录、权限、模型配置、记忆治理等核心判断在后端完成，减少前端硬编码风险。
4. **企业级可审计**：调用链路、工具结果、耗时、用户与租户上下文可追踪。
5. **面向 SaaS 商业化**：多租户、用户上限、租户到期、API Key、用量分析为商业运营预留基础。

## 8. 客户收益

- **降本**：减少每个 AI 应用重复开发工具接入与权限治理的成本。
- **提效**：业务 API 可快速变成 Agent 工具，缩短 AI 应用上线周期。
- **控险**：用租户隔离、角色权限、审计日志降低 AI 越权和黑盒调用风险。
- **可扩展**：从单个部门试点扩展到多租户、多系统、多 Agent 协同。

## 9. 实施路径

1. **需求梳理**：确定首批接入系统、用户角色、AI 助手场景。
2. **接口接入**：配置连接器，导入 OpenAPI，生成 MCP 工具。
3. **权限配置**：创建租户角色，配置功能菜单和工具权限。
4. **模型配置**：接入企业可用模型供应商，配置默认模型。
5. **Skill 编排**：将高频业务流程沉淀为 Skill。
6. **试点运营**：通过审计日志、用量分析、用户反馈优化能力边界。
7. **规模推广**：扩展更多系统、租户、部门和 Agent 场景。

## 11. 商业化适配

EASP 面向 B2B SaaS 产品化交付，支持：

- 按租户开通与到期控制。
- 按用户上限、工具数量、模型用量、Skill 执行次数做套餐设计。
- 为企业客户提供私有化、专属模型供应商、专属连接器等扩展。
- 通过 API Key 与 MCP 接入文档支持外部 Agent/系统集成。

## 12. 下一步建议

- 选择 1-2 个高价值业务流程作为试点，如“客服工单自动处理”或“销售客户查询助手”。
- 接入首批 3-5 个核心 API，并配置角色权限。
- 使用审计与用量数据评估上线效果。
- 逐步将高频操作沉淀为 Skill，把经验沉淀为可治理记忆。
