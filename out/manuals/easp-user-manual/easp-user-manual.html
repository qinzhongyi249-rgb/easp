5. 租户、用户与角色管理

创建租户

  1. 进入「租户管理」。
  2. 点击「新建租户」。
  3. 填写租户名称、管理员邮箱、初始密码、到期时间、用户上限。
  4. 保存并通知租户管理员。

创建用户

  1. 进入「用户管理」。
  2. 点击「新建用户」。
  3. 填写邮箱、姓名、初始密码、状态。
  4. 保存后分配角色。

配置角色权限

  1. 进入「角色管理」。
  2. 创建角色并填写描述。
  3. 勾选功能权限。
  4. 分配给用户。

删除与恢复

用户删除为软删除；误删后可通过恢复入口恢复。创建和注册会校验租户状态、到期和用户上限。

6. 功能权限说明

功能权限控制左侧菜单和操作入口。可配置项包括：用户管理、角色管理、连接器、MCP 工具、技能管理、记忆管理、模型配置、SSO 配置、AI 助手、用量分析、审计日志、API Key。

特殊入口说明
dashboard基础入口，通常不纳入角色功能权限配置。
tenants系统管理员专属，不纳入租户角色权限配置。
*表示拥有全部菜单权限。
如果用户看不到菜单，优先检查用户角色和角色 tools 功能权限。

7. 模型配置

进入「模型配置」。
新增模型供应商，填写名称、Base URL、认证信息等。
新增模型配置，设置模型名称、显示名、启用状态。
选择稳定可用的模型设为默认。
在 AI 助手或模型调用中验证回复。
安全提示：不要在普通文档、聊天或截图中传播 API Key。模型不可用时应明确报错，不应静默 fallback 到错误配置。

8. 连接器与 MCP 工具

创建连接器

  1. 进入「连接器」。
  2. 点击「新建连接器」。
  3. 填写名称、传输类型、目标地址、认证头。
  4. 保存并测试连接。

生成 MCP 工具

  1. 进入「MCP 工具」。
  2. 导入 OpenAPI 文档。
  3. 检查工具名称、描述、参数 Schema。
  4. 启用工具并测试调用。
预期结果：AI 助手和外部 Agent 可在授权范围内通过标准 MCP 协议调用企业 API。

9. Skill 管理与执行

创建 Skill

  1. 进入「技能管理」。
  2. 填写名称、描述、分类、标签。
  3. 配置输入/输出 Schema。
  4. 添加执行步骤，步骤可调用 MCP 工具。

执行 Skill

  1. 选择目标 Skill。
  2. 根据 Schema 表单填写参数。
  3. 点击执行。
  4. 查看 Timeline、步骤耗时和输出。

最佳实践：优先将高频、稳定、可标准化的业务流程沉淀为 Skill。

10. AI 助手使用

独立助手页面

  1. 进入「AI 助手」。
  2. 输入自然语言任务。
  3. 助手根据权限选择可用工具。
  4. 查看实时回复、工具调用过程和最终结果。

悬浮助手

可结合当前页面上下文回答问题或执行操作。默认显示一个实时过程气泡，展开后查看完整 TraceTimeline。

AI 助手只能使用当前用户被授权的工具和功能。

11. 记忆管理

治理配置

  • 自动提取开关
  • 召回开关
  • 敏感过滤
  • 记忆审计

召回解释

查看 keyword、vector、recency、frequency、type 等分数，理解记忆为何命中。

长期记忆

user_memories 用于长期用户记忆、召回注入和用户记忆页展示。

记忆池条目

memory_pools + memory_entries 用于 AI 助手记忆池工具保存的池条目。

排查提示：如果 AI 助手能回答某条记忆但页面看不到，应同时检查长期记忆与记忆池条目。

12. SSO、用量、审计与 API Key

SSO 配置

  1. 进入「SSO 配置」。
  2. 填写身份提供商配置。
  3. 测试连接。
  4. 获取租户登录 URL。

用量分析

查看 API 调用量、Token 消耗、配额使用和功能来源统计。

审计日志

追踪用户操作、AI 工具调用、Skill 执行、记忆治理动作，包含 Agent ID、耗时、IP、User-Agent 等上下文。

API Key

创建外部访问密钥，查看公开接入文档 /docs/api-key-access，并定期轮换或禁用。

13. 常见问题与排查

问题优先检查
用户看不到菜单用户角色、角色 tools 功能权限、是否为系统管理员专属入口
AI 助手无法调用工具工具是否启用、用户权限、连接器状态、模型配置
模型回复报错模型供应商、默认模型、审计日志、后端日志
记忆管理页和助手回答不一致长期记忆与记忆池条目两个来源
OpenAPI 导入后工具不可用接口地址、认证头、参数 Schema、响应格式、网络连通性

14. 运维与安全建议

  • 健康检查：/health。
  • 外网入口通常由 nginx 转发到后端服务。
  • 前端/API 入口端口：8080；后端服务端口：8082。
  • 账号、密码、API Key、连接串不应写入对外手册。
  • 高风险工具上线前先禁用或仅对小范围角色开放。
  • 定期查看审计日志和用量分析，优化成本与安全边界。
EASP 使用原则：先配置身份与权限，再接入工具和模型，最后通过审计、用量和记忆治理持续优化。
