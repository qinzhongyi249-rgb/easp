# EASP 使用手册

> Enterprise AI Service Platform｜企业级 API-to-MCP 智能服务平台  
> 版本：V1.0｜适用对象：系统管理员、租户管理员、运营人员、开发/集成人员、业务用户

## 1. 手册说明

本手册用于指导企业用户完成 EASP 平台的日常使用，包括登录、租户与用户管理、角色权限配置、连接器接入、MCP 工具管理、Skill 执行、记忆管理、模型配置、AI 助手、用量分析、审计日志与 API Key 管理。

> 访问入口：以部署环境为准。当前临时外网入口可使用 `http://8.130.48.99:8080/`。账号、密码与租户信息请向平台管理员获取。

## 2. 用户角色与权限边界

| 角色类型 | 典型职责 | 可操作范围 |
|---|---|---|
| 系统管理员 | 管理租户、系统级角色、全局配置 | 租户管理、跨租户运维、系统初始化 |
| 租户管理员 | 管理本租户用户、角色、权限和配置 | 用户、角色、连接器、模型、SSO、审计等 |
| 运营/业务管理员 | 管理业务工具、Skill、记忆与用量 | MCP 工具、Skill、AI 助手、用量分析 |
| 普通业务用户 | 使用已授权的 AI 能力 | AI 助手、可见工具、已授权页面 |

权限原则：EASP 采用功能菜单权限与后端权限校验结合的方式。前端只展示可见入口，后端负责核心决策和安全校验。

## 3. 快速开始

### 3.1 登录平台

1. 打开平台访问地址。
2. 输入管理员分配的邮箱、密码与租户信息。
3. 如企业启用 SSO，可通过租户 SSO 登录入口进入。
4. 登录成功后进入仪表盘。

预期结果：左侧显示当前账号可访问的功能菜单。

常见问题：
- 看不到某个菜单：联系租户管理员检查角色的 `tools` 功能权限。
- 登录失败：检查租户是否启用、是否到期、账号是否被停用或删除。

### 3.2 推荐初始化流程

1. 系统管理员创建租户。
2. 租户管理员创建用户与角色。
3. 配置角色功能权限。
4. 配置模型供应商与默认模型。
5. 创建连接器并导入 OpenAPI。
6. 生成并测试 MCP 工具。
7. 创建 Skill 或在 AI 助手中调用工具。
8. 开启审计与用量观察，逐步推广。

## 5. 租户管理

适用对象：系统管理员。

### 5.1 创建租户

1. 进入「租户管理」。
2. 点击「新建租户」。
3. 填写租户名称、管理员邮箱、管理员初始密码、到期时间、用户上限等信息。
4. 保存。

预期结果：系统创建租户，并初始化租户管理员账号。

注意事项：
- `expires_at` 为空表示永久有效。
- `max_users=0` 表示不限制用户数。
- 租户停用或到期后，登录和注册会被阻止。

## 6. 用户管理

适用对象：租户管理员。

### 6.1 创建用户

1. 进入「用户管理」。
2. 点击「新建用户」。
3. 填写邮箱、姓名、初始密码、状态等信息。
4. 保存后，为用户分配角色。

### 6.2 删除与恢复用户

- 删除用户为软删除，不会立即物理删除数据。
- 如误删，可在支持恢复的入口执行恢复操作。

注意事项：注册与创建用户时，系统会校验租户是否存在、是否启用、是否到期、用户数是否超限，避免产生孤立用户。

## 7. 角色与功能权限

适用对象：租户管理员。

### 7.1 创建角色

1. 进入「角色管理」。
2. 点击「新建角色」。
3. 填写角色名称和描述。
4. 在「功能权限」中选择该角色可访问的菜单。
5. 保存后分配给用户。

### 7.2 功能权限说明

可配置的功能权限包括：用户管理、角色管理、连接器、MCP 工具、技能管理、记忆管理、模型配置、SSO 配置、AI 助手、用量分析、审计日志、API Key。

说明：
- `dashboard` 是基础入口，通常不纳入角色功能权限配置。
- `tenants` 是系统管理员专属入口，不纳入租户角色权限配置。
- `*` 表示拥有全部菜单权限。

## 8. 模型配置

适用对象：租户管理员、平台运维。

### 8.1 添加模型供应商

1. 进入「模型配置」。
2. 新增模型供应商，填写供应商名称、Base URL、API Key 等配置。
3. 保存后新增模型配置。

### 8.2 设置默认模型

1. 在模型列表中选择可用模型。
2. 点击「设为默认」。
3. 确认默认模型生效。

注意事项：EASP 不应静默 fallback 到错误模型配置。模型不可用时应明确报错，便于管理员排查。

## 9. 连接器管理

适用对象：开发/集成人员、租户管理员。

### 9.1 创建连接器

1. 进入「连接器」。
2. 点击「新建连接器」。
3. 填写连接器名称、协议/传输类型、目标地址、认证头等信息。
4. 保存并测试连接。

安全建议：不要在说明字段中明文记录密钥、Token 或连接串。凭证应放入专用配置字段并受权限保护。

## 10. MCP 工具管理

适用对象：开发/集成人员、运营人员。

### 10.1 导入 OpenAPI 并生成工具

1. 进入「MCP 工具」。
2. 选择导入 OpenAPI 文档。
3. 系统解析接口并生成工具定义。
4. 检查工具名称、描述、参数 Schema 和响应映射。
5. 启用工具并执行测试调用。

预期结果：AI 助手和外部 Agent 可在授权范围内调用这些 MCP 工具。

### 10.2 工具启停

- 对暂不可用或风险较高的工具，可先禁用。
- 调整工具参数后应重新测试。

## 11. Skill 管理

适用对象：运营人员、业务管理员、开发人员。

### 11.1 创建 Skill

1. 进入「技能管理」。
2. 点击「新建 Skill」。
3. 填写名称、描述、分类、标签、输入 Schema、输出 Schema。
4. 添加步骤，步骤可以调用 MCP 工具或执行内置动作。
5. 保存后进入测试执行。

### 11.2 执行 Skill

1. 选择目标 Skill。
2. 根据 Schema 表单填写输入参数。
3. 点击执行。
4. 查看 Timeline 执行结果、每个步骤耗时和输出。

最佳实践：优先将高频、稳定、可标准化的业务流程沉淀为 Skill。

## 12. AI 助手使用

适用对象：所有授权用户。

### 12.1 独立 AI 助手页面

1. 进入「AI 助手」。
2. 输入自然语言问题或任务。
3. 助手会根据用户权限选择可用工具。
4. 查看实时回复、工具调用过程和最终结果。

### 12.2 悬浮 AI 助手

- 悬浮助手可结合当前页面上下文回答问题或执行操作。
- 默认只显示一个实时过程气泡，展开后可查看完整 TraceTimeline。

注意事项：AI 助手只能使用当前用户被授权的工具和功能。

## 13. 记忆管理

适用对象：租户管理员、运营人员。

### 13.1 记忆治理配置

1. 进入「记忆管理」。
2. 配置自动提取、召回、敏感过滤、审计等开关。
3. 查看召回解释，理解记忆为何被命中。
4. 查看记忆审计日志，排查写入、去重、脱敏、归档等动作。

### 13.2 记忆池与长期记忆

- `user_memories`：长期用户记忆，用于召回注入和用户记忆页展示。
- `memory_pools + memory_entries`：AI 助手记忆池工具保存的池条目。

排查提示：如果 AI 助手能回答某条记忆但页面看不到，应同时检查长期记忆与记忆池条目两个视图。

## 14. SSO 配置

适用对象：租户管理员、企业 IT。

1. 进入「SSO 配置」。
2. 填写身份提供商配置。
3. 测试连接。
4. 获取租户登录 URL。
5. 通知企业用户使用 SSO 登录。

说明：租户登录接口支持 SSO 与标准登录自动回退，前端无需硬编码判断登录路径。

## 15. 用量分析

适用对象：租户管理员、运营人员。

可查看：
- API 调用量。
- Token 消耗。
- 配额使用情况。
- 不同功能来源的用量统计。

建议：定期复盘高消耗工具、失败率高的接口和高频 Skill，优化成本与稳定性。

## 16. 审计日志

适用对象：租户管理员、安全/合规人员。

审计日志用于追踪：
- 用户操作。
- AI 助手工具调用。
- Skill 执行。
- 记忆写入、去重、脱敏、归档等治理动作。

详情字段通常包括：操作人、租户、目标对象、Agent ID、耗时、IP、User-Agent、执行结果等。

## 17. API Key 管理

适用对象：租户管理员、开发/集成人员。

1. 进入「API Key」。
2. 创建访问密钥。
3. 按业务系统或 Agent 分配用途和权限。
4. 查看公开接入文档 `/docs/api-key-access`。
5. 定期轮换或禁用不再使用的 Key。

安全建议：API Key 只展示一次时应立即妥善保存，不要通过聊天、文档或截图传播密钥。

## 18. 常见问题

### Q1：用户看不到菜单怎么办？
检查用户是否拥有对应角色，以及角色的功能权限是否包含目标菜单 key。

### Q2：AI 助手无法调用某个工具怎么办？
检查工具是否启用、用户是否有权限、连接器是否可用、模型配置是否正常。

### Q3：模型回复报错怎么办？
优先查看模型配置、供应商状态、审计日志和后端日志。平台应明确报错，不应静默切换到错误配置。

### Q4：记忆管理页和助手回答不一致怎么办？
同时检查长期记忆与记忆池条目。两类存储用途不同，页面需展示完整来源。

### Q5：导入 OpenAPI 后工具不可用怎么办？
检查接口地址、认证头、参数 Schema、响应格式、网络连通性和工具启用状态。

## 19. 运维与支持

- 健康检查：`/health`。
- 外网入口通常由 nginx 转发到后端服务。
- 后端服务端口：`8082`；前端/API 入口端口：`8080`。
- 具体环境、账号和密钥由管理员维护，不应写入对外手册。

**EASP 使用原则：先配置身份与权限，再接入工具和模型，最后通过审计、用量和记忆治理持续优化。**
